Google Apps Script Setup Guide

Fountain Onboarding Form Builder

1 Create the Apps Script Project

  1. Go to script.google.com
  2. Click New Project
  3. Delete the default code in Code.gs
  4. Copy the entire contents of apps-script.js and paste it
  5. Click File > Save (Ctrl+S), name it "Fountain Onboarding"

2 Configure Your Settings

Find the CONFIG section at the top (~line 36) and update:

const CONFIG = {
  // Your Google Sheet ID (from the URL)
  SPREADSHEET_ID: 'your-spreadsheet-id-here',

  // Emails for new hire notifications
  HR_EMAILS: ['hr@yourcompany.com'],

  // Your Vercel app URL
  APP_URL: 'https://your-app.vercel.app',

  // Leave empty to auto-create folder
  UPLOADS_FOLDER_ID: '',

  HYPERLINK_TEXT: 'View File'
};

Finding Your Spreadsheet ID

Open your Google Sheet and copy the ID from the URL:

https://docs.google.com/spreadsheets/d/THIS_IS_YOUR_ID/edit

3 Authorize the Script

  1. In the Apps Script editor, select authorizeAndTest from the dropdown
  2. Click Run
  3. Click Review Permissions in the popup
  4. Select your Google account
  5. Click Advanced > Go to Fountain Onboarding (unsafe)
  6. Click Allow

4 Deploy as Web App

  1. Click Deploy > New deployment
  2. Click the gear icon, select Web app
  3. Configure:
    • Execute as: Me
    • Who has access: Anyone
  4. Click Deploy
  5. Copy the Web App URL

5 Add URL to Your App

Add the Web App URL to your Vercel environment variables:

APPS_SCRIPT_WEB_APP_URL=https://script.google.com/macros/s/xxxxx/exec

6 Test Your Setup

Run these functions in the Apps Script editor to verify:

Function Purpose
validateConfig() Check all settings are configured
testSpreadsheet() Add a test row to your sheet
testEmail() Send test confirmation emails
testFileUpload() Upload a test file to Drive

Troubleshooting

Issue Solution
Permission denied Re-run authorizeAndTest()
Emails not sending Verify HR_EMAILS has valid addresses
Files not uploading Run getUploadsFolderInfo() to check access
Spreadsheet errors Verify SPREADSHEET_ID is correct
Need the uploads folder ID? Run getUploadsFolderInfo() after the first file upload to get the auto-created folder's ID.
